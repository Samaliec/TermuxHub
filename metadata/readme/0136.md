# Gembot Yal'Lilith — Resurrection from Ashes Trigger for Termux

> Safe local-first bootstrap for Termux. It is **not** a “dead hand” automation clone, does **not** attempt covert persistence, and does **not** bypass Android / Termux restrictions. Instead, it gives you a recoverable trigger that validates itself before each start, respects quiet hours, and restores a last-known-good runtime if the local assistant files break.

## What this recipe does

- Creates a **Termux-friendly launch trigger** for `Gembot Yal'Lilith`.
- Uses a **deterministic self-test** before each start.
- Respects a configurable **quiet hours** window (`QUIET_HOURS_START` / `QUIET_HOURS_END`).
- Restores the assistant from a **last-known-good** copy when validation fails.
- Writes a local `heartbeat.json` with node name, local IPv4 addresses, allowed local CIDRs, and an execution sequence number.
- Keeps control **inside the current user space only** — no root, no hidden privilege escalation, no external command-and-control.

## Install in Termux

```bash
pkg update -y
pkg install -y bash coreutils cronie python termux-services curl
mkdir -p ~/.local/share/gembot-yal-lilith/bin
curl -fsSL https://raw.githubusercontent.com/maazm7d/TermuxHub/main/metadata/assets/0136/resurrection-trigger.sh -o ~/.local/share/gembot-yal-lilith/bin/resurrection-trigger.sh
curl -fsSL https://raw.githubusercontent.com/maazm7d/TermuxHub/main/metadata/assets/0136/assistant_stub.py -o ~/.local/share/gembot-yal-lilith/bin/assistant_stub.py
chmod +x ~/.local/share/gembot-yal-lilith/bin/resurrection-trigger.sh ~/.local/share/gembot-yal-lilith/bin/assistant_stub.py
bash ~/.local/share/gembot-yal-lilith/bin/resurrection-trigger.sh --install
crond
bash ~/.local/share/gembot-yal-lilith/bin/resurrection-trigger.sh --cycle
```

## Trigger workflow

1. `--install`
   - Creates the state, log, config, and backup directories.
   - Saves the current trigger and assistant stub to `last-known-good/`.
   - Installs a cron cycle that runs every 5 minutes.
2. `--cycle`
   - Refuses concurrent cycles with a lock directory.
   - Validates dependencies.
   - Checks whether the current time falls into quiet hours.
   - Runs `python assistant_stub.py --self-test`.
   - If validation fails, restores the last-known-good runtime.
   - Starts the assistant only when tests pass and quiet hours are inactive.
3. `--status`
   - Shows whether the assistant is running and whether quiet hours are active.
4. `--stop`
   - Stops the current assistant process for maintenance.

## Config file

After installation, edit:

```bash
nano ~/.config/gembot-yal-lilith/trigger.env
```

Defaults:

```bash
QUIET_HOURS_START="00:30"
QUIET_HOURS_END="06:30"
ALLOWED_LOCAL_CIDRS="127.0.0.1/32,192.168.0.0/16,10.0.0.0/8"
HEARTBEAT_NAME="Fractal Node v6"
ASSISTANT_CMD=(python "$HOME/.local/share/gembot-yal-lilith/bin/assistant_stub.py")
SELF_TEST_CMD=(python "$HOME/.local/share/gembot-yal-lilith/bin/assistant_stub.py" --self-test)
```

## How to replace the demo stub with your own assistant

Replace only these two variables in `trigger.env`:

```bash
ASSISTANT_CMD=(python "$HOME/path/to/your_assistant.py")
SELF_TEST_CMD=(python "$HOME/path/to/your_assistant.py" --self-test)
```

Recommendations:

- Bind any local API to **loopback or your trusted LAN only**.
- Keep a **deterministic `--self-test` mode** that exits with code `0` only when the assistant is safe to launch.
- Make your assistant write its own health / heartbeat state to the same `GEMBOT_STATE_DIR`.
- Keep a **recoverable artifact** in `last-known-good/` rather than trying to regenerate unknown code automatically.

## Notes about quiet hours in Termux

Android may still throttle or defer work when the device is idle, under battery optimizations, or after reboot. This recipe does not try to bypass those platform limits. Instead, it ensures the trigger stays **predictable**, **recoverable**, and **auditable** inside the constraints that Termux allows.
